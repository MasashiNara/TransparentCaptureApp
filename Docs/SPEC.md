# 透明キャプチャ文字起こしアプリ 仕様書

## 1. 目的

Windows 11 上で、画面の任意範囲を透明オーバーレイ越しに指定し、その範囲をキャプチャして LLM に送信し、画像内の文字をテキスト化して保存するデスクトップアプリを作成する。

## 2. 対象環境

- OS: Windows 11
- アプリ種別: デスクトップアプリ
- 推奨実装: C# / .NET / WPF
- 開発環境: Visual Studio Community 2022、または .NET SDK + VS Code

## 3. 基本コンセプト

アプリは常に前面に表示できる透明ウィンドウとして動作する。

ウィンドウには以下のみを表示する。

- 外枠
- 上部 1 行分の操作バー
- 操作バー上の 4 つのボタン

操作バー以外の内側領域は透明で、背後の画面が透けて見える。この透明領域をキャプチャ対象範囲とする。

## 4. 主要機能

### 4.1 メインウィンドウ

メインウィンドウは以下を備える。

- 透明なキャプチャ対象領域
- キャプチャ対象領域を示す枠線
- 上部操作バー
- ウィンドウ移動
- ウィンドウサイズ変更
- 常に最前面表示

初期表示サイズは幅 900 px、高さ 500 px とする。

ウィンドウは常に最前面に固定し、初期版では設定による無効化は行わない。

### 4.2 操作ボタン

操作バーには以下のボタンを左から配置する。

- 設定
- キャプチャ
- ログ
- 保存先を開く

### 4.3 キャプチャ機能

「キャプチャ」ボタン押下時、以下を実行する。

1. 現在の透明領域の画面座標を取得する。
2. その座標範囲にある背後の画面をスクリーンショットとして取得する。
3. 画像ファイルとして保存する。
4. 画像を LLM に送信する。
5. LLM の応答から文字起こしテキストを取得する。
6. テキストファイルとして保存する。
7. 設定に応じて、保存したテキストファイルを既定のテキストエディタで開く。
8. 処理結果をログに記録する。

キャプチャ対象は透明領域全体とする。アプリ自身の操作バーや枠線は含めない。

複数モニター環境では、透明領域が存在する仮想スクリーン座標を元にキャプチャ範囲を計算する。

### 4.4 LLM 文字起こし機能

画像入力対応の LLM API にキャプチャ画像を送信し、画像内の文字をテキスト化する。

初期方針:

- LLM プロバイダは設定画面で選択可能にする。
- 対応プロバイダは OpenAI API、Anthropic API、Ollama、llama.cpp server の 4 種類とする。
- ChatGPT Pro のログイン認証は利用しない。
- OpenAI API と Anthropic API はユーザーが API キーを入力する。
- Ollama はローカルまたは指定 URL の Ollama サーバーへ接続する。
- llama.cpp server はローカルまたは指定 URL の OpenAI 互換 API へ接続する。
- モデル名はプロバイダごとに設定可能にする。
- プロバイダごとの初期モデル名は以下とする。
  - OpenAI API: `gpt-5.4-mini`
  - Anthropic API: `claude-sonnet-4-20250514`
  - Ollama: `gemma3`
  - llama.cpp server: `llama.cpp`
- API キー保存方式: DPAPI によるローカル暗号化を第一候補とする。
- プロンプト内容は設定画面で編集可能にする。

初期プロンプト:

```text
画像内に含まれる文字をできるだけ正確に文字起こししてください。
レイアウトが分かる範囲で改行を維持してください。
説明や補足は追加せず、文字起こし結果のみを返してください。
```

### 4.5 設定機能

「設定」ボタン押下で設定画面を開く。

設定画面では以下を変更できる。

- 保存先フォルダ
- LLM プロバイダ
- OpenAI API キー
- OpenAI モデル名
- Anthropic API キー
- Anthropic モデル名
- Ollama 接続 URL
- Ollama モデル名
- llama.cpp 接続 URL
- llama.cpp モデル名
- llama.cpp 画像入力対応チェック
- 文字起こしプロンプト
- キャプチャ画像を保存するか
- 文字起こし成功後にキャプチャ画像を削除するか
- テキストファイルを保存するか
- テキスト保存後に自動で開くか
- ログファイルの保存先

設定値はローカルに保存し、次回起動時に復元する。

llama.cpp 画像入力対応チェックでは、設定された llama.cpp 接続 URL の `GET /props` を呼び出し、応答内の `modalities.vision` が `true` か確認する。

モデル名が入力されている場合は、router mode を考慮して `GET /props?model=(model_name)` を優先して確認する。

### 4.6 ログ機能

アプリの動作ログを `.txt` 形式で保存する。

ログに記録する内容:

- アプリ起動
- 設定変更
- キャプチャ開始
- キャプチャ画像保存先
- LLM 送信開始
- LLM 応答成功
- テキスト保存先
- エラー内容

「ログ」ボタン押下時、ログファイルを既定のテキストエディタで開く。

### 4.7 保存先を開く

「保存先を開く」ボタン押下時、設定された保存先フォルダを Windows Explorer で開く。

保存先フォルダが存在しない場合は作成する。

## 5. ファイル保存仕様

### 5.1 保存先

ユーザーが設定画面で指定したフォルダに保存する。

未設定時の既定保存先:

- `Documents\TransparentCapture`

### 5.2 キャプチャ画像

ファイル形式:

- PNG

ファイル名:

```text
capture_yyyyMMdd_HHmmss.png
```

### 5.3 文字起こしテキスト

ファイル形式:

- TXT

ファイル名:

```text
transcript_yyyyMMdd_HHmmss.txt
```

文字コード:

- UTF-8

### 5.4 ログファイル

ファイル形式:

- TXT

ファイル名:

```text
app.log.txt
```

## 6. エラー処理

以下の場合はログに記録し、ユーザーに簡潔なエラーメッセージを表示する。

- 保存先フォルダに書き込めない
- キャプチャに失敗した
- API キーが未設定
- Ollama サーバーに接続できない
- llama.cpp server に接続できない
- LLM API 呼び出しに失敗した
- 通信タイムアウト
- 応答から文字起こし結果を取得できない

## 7. セキュリティ・プライバシー

キャプチャ画像には機密情報が含まれる可能性があるため、以下を考慮する。

- API キーは平文保存を避ける。
- 可能であれば Windows 資格情報マネージャー、または DPAPI を利用する。
- キャプチャボタン押下後は確認ダイアログなしで LLM に送信する。
- キャプチャ画像を保存しない設定を用意する。
- 文字起こし成功後にキャプチャ画像を削除する設定を用意する。

## 8. 非機能要件

- 通常操作が軽いこと
- キャプチャ処理中に UI が固まらないこと
- 複数モニター環境に対応すること
- Windows の DPI スケーリングを考慮すること
- ログにより失敗原因を追跡できること

## 9. MVP 範囲

最初の実装対象は以下とする。

1. 透明オーバーレイウィンドウ
2. 4 ボタンの操作バー
3. 保存先設定
4. 透明領域のスクリーンショット保存
5. ログファイル出力
6. ログファイルを開く
7. 保存先フォルダを Explorer で開く
8. LLM API による文字起こし
9. 文字起こし結果の TXT 保存
10. 設定に応じた TXT 自動表示

## 11. 要確認事項

現時点で実装前に必須で確認すべき事項はなし。

## 12. 確定事項

- ChatGPT Pro の契約認証はアプリから利用しない。
- LLM プロバイダは OpenAI API、Anthropic API、Ollama、llama.cpp server から選択できるようにする。
- API キーはユーザーが設定画面で入力する。
- 初期保存先は `Documents\TransparentCapture` とする。
- キャプチャ画像は基本的に保存する。
- 文字起こし成功後にキャプチャ画像を削除する設定を用意する。
- キャプチャボタン押下後、送信前確認ダイアログは表示しない。
- キャプチャ範囲は透明領域全体とする。
- ウィンドウは常に最前面に固定する。
- 複数モニター環境は初期版から必須対応とする。
- プロンプト文は設定画面で編集可能にする。
- テキスト保存後はデフォルトで自動表示する。
- テキスト自動表示は設定画面で無効化できる。
- 配布形式は初期版では単体 exe とする。
- インストーラーは必要になった段階で検討する。

## 13. 配布仕様

初期版は単体 exe として配布する。

インストーラー、スタートメニュー登録、自動アップデート、アンインストーラーは初期版では対象外とする。
